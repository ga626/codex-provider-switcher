# CodeX Provider Switcher 0.4.0-alpha 发布说明

## 摘要

本版本为旧工具替换建立受控迁移和单一写入者边界。它不自动停止、删除或覆盖旧版工具；最终替换仍需在新会话中按交接手册执行。

## 主要改动

- 新增“交接准备”工作区，展示旧工具进程、旧端口、导入状态和切换门禁。
- 旧 profiles 改为显式预检和导入：只显示条数、格式和冲突数，不显示 API 密钥。
- 导入不覆盖新版已有非占位配置，不修改旧 profiles 文件。
- 旧工具仍运行或占用旧端口时，真实切换会被阻止，避免两个工具同时写 Codex 配置。
- 切换前备份新增不含凭据内容的 manifest。
- GitHub CI 只在 PR 和主线合并后运行完整验证，功能分支 push 不重复触发。

## 用户影响

- 用户仍无需更新私钥或口令；签名私钥只在 GitHub Actions 发布新 tag 时使用。
- 先在新版导入并验证服务商，再由新 Codex 会话完成最终交接。
- 旧工具目录保留为回滚参考，任何失败都应恢复备份并重启旧工具。

## 发布后动作

1. 从 GitHub Release 下载 setup 并验证安装、启动、关闭和无 CMD 常驻。
2. 从现有稳定安装执行一次到 `0.4.0-alpha` 的真实应用内更新验收，确认签名、替换、重启和用户数据保留。
3. 由新会话按 [旧工具交接手册](legacy-cutover-handoff.zh.md) 决定是否执行实际替换。
