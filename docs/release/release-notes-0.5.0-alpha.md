# CodeX Provider Switcher 0.5.0-alpha 发布说明

## 摘要

本版本修复已发现新版本后，点击下载仍被手动 Release 页地址校验阻断的问题，并完善服务商真实验证与模型选择的可信边界。

## 主要改动

- 已由 Tauri updater 校验并准备好的更新，直接执行签名下载、安装和重启；手动打开下载页的 fallback 仍限制为本项目 GitHub Release 页面。
- 兼容性探测由用户显式触发，以当前模型发送短时的已认证 `/v1/responses` 请求；成功才记录当前模型的正向证据。
- 切换不再自动发起远端探测。探测未确认不会阻止本地安全切换，也不等同于服务商不能被 Codex 使用。
- `/v1/models` 继续只用于读取模型目录。界面只展示服务商目录与当前模型的正向探测结果，不再给所有模型贴“未验证”。
- 手动输入且不在最近目录中的模型，保存前需要二次确认；确认和探测均不替代实际 Codex 使用结果。

## 用户影响

`0.3.2-alpha` 和 `0.4.0-alpha` 包含更新按钮阻断问题，无法通过自身自动升级到本版本。请从 GitHub Release 手动运行一次 `0.5.0-alpha` setup。用户不需要输入签名私钥、口令或 Release 地址。

`0.5.0-alpha` 是修复后的自动更新基线。下一个高于 `0.5.0-alpha` 的版本发布时，才从已安装的 `0.5.0-alpha` 进行真实的下载、签名校验、重启、版本变化和用户数据保留验收。

## 发布后验收

1. 从 GitHub Release 下载 setup，并校验 SHA256。
2. 覆盖安装到 `D:\Software\CodeX Provider Switcher` 后，确认打开一个桌面窗口、没有常驻 CMD 或外部浏览器。
3. 确认模型目录只展示服务商目录和当前模型的正向探测结果；选择或手动保存模型后，运行兼容性探测不会写入 Codex 配置或凭据。
4. 保持旧工具不变。本版本不执行最终 provider cutover。
