# 旧工具交接手册

## 目的

本手册用于把旧版 `CodeX-Switcher.exe` 的实际使用权交给 `CodeX Provider Switcher`。这不是两个工具的长期共存方案，也不是当前开发会话可以直接执行的操作。

## 前提

- 已安装包含迁移门禁的新 Release。
- 新版“交接准备”显示旧 profiles 已导入，且所有要使用的服务商已完成真实验证。
- 新版能显示备份和恢复入口。
- 旧工具目录、旧 profiles 和旧程序文件保持原样，只作为回滚参考。
- 由新 Codex 会话或另一个 agent 执行，不使用刚改完切换逻辑的当前会话。

## 切换步骤

1. 记录脱敏状态：新版版本、旧工具进程/端口状态、导入状态、最近备份数量。
2. 创建一份新的切换前备份，确认 manifest 只列出文件名而不含配置内容或密钥。
3. 停止旧 `CodeX-Switcher.exe`，确认旧端口已经释放。
4. 从安装版启动新版桌面应用，确认窗口正常、没有常驻命令行窗口、没有自动打开浏览器。
5. 选择已验证的已导入服务商，执行一次真实切换。
6. 确认 `model_provider = "custom"`、`wire_api = "responses"`、`disable_response_storage = true`，并确认受保护配置段仍存在。不要把配置内容粘贴进记录。
7. 在新会话运行一次最小 Codex smoke，确认用户实际入口可用。
8. 成功后把旧工具标记为回滚参考，不删除目录或 profiles。

## 失败回滚

任一环节失败时立即停止继续写入：恢复最近备份，重新启动旧工具，确认其端口恢复监听，并把状态记录为“新版代码已发布，产品替换未完成”。不要在故障会话中反复尝试切换。

## 发布验收

该交接属于发布后动作。发布前必须从 GitHub Release 下载新 setup，验证签名更新资产，并从既有稳定安装执行一次真实跨版本升级，确认程序文件更新而 `%LOCALAPPDATA%\CodeX Provider Switcher` 中的用户数据仍保留。
