# 旧工具替换手册

这份手册只面向维护者和执行最终交接的新 Codex 会话。它处理的是把旧 `CodeX-Switcher.exe` 退到回滚位，不是给产品界面增加“交接功能”。

## 什么时候可以开始

同时满足以下条件前，不要停止旧工具：

1. 新版本已经从 Microsoft Store 安装，并按普通用户路径启动和验证。Store 认证尚未完成时，只能使用维护者本机候选安装做继续使用与验收，不能开始最终替换。
2. 新安装版已完成更新验收，且用户资料在升级后仍存在。
3. 目标 provider 已在新工具中通过“服务商可用性测试”。
4. 当前 Codex 会话已经结束；准备使用新的会话完成最终确认。
5. `npm run qa:cutover-preflight` 的只读结果没有意外自启来源、安装缺失或配置不变量缺失。

## 固定顺序

1. 打开新的 Codex 会话，不要在原会话中切 provider。
2. 运行：

   ```powershell
   npm run qa:cutover-preflight
   ```

   记录新安装版版本和签名状态、旧程序 PID/监听端口、Run/RunOnce、Startup、计划任务、服务，以及 Codex 配置不变量。脚本不打开 `auth.json`，不打印任何凭据，也不会停止任何进程。

3. 从新安装版启动 `Signalman AI`，确认没有 CMD、浏览器或后台托盘。确认已经由 Store 版接管后，才卸载维护者本机候选安装；保留已有的本机用户资料。
4. 确认目标 provider 的测试通过后执行切换。应用必须先创建恢复点。
5. 在新的 Codex 会话中完成一次真实工作确认。
6. 只有上一步成功，才停止旧 `CodeX-Switcher.exe`。不要删除旧目录或旧 `profiles.json`。
7. 重启 Windows，再运行一次 preflight。确认旧工具没有重启、47831 不再监听，且新的工作流正常。

## 出现问题时

- 新工具无法启动、测试不通过或切换后异常：不要停止旧工具；使用新工具的恢复点，保留脱敏错误信息。
- 旧工具在重启后又出现：preflight 会显示启动来源。先禁用准确来源，再次重启复查；不要盲目删除文件。
- 新版本没有可信 Windows 代码签名、Release 资产不完整或更新失败：状态是“代码已合并，产品未交付”。等待受控发布，不进行替换。

## 完成标准

最终收口记录必须写清：新版本/tag、Release 链接、下载与安装结果、更新结果、provider 测试结果、真实新会话验收结果、停止旧进程结果、重启后的自启复查结果与回滚位置。缺任何一项，旧工具仍是后备，不称为“已替换”。
